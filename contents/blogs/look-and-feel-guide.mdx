## Why a Consistent Look & Feel Matters

A unified and consistent Look and Feel is essential for creating a cohesive and professional digital product. In Pressbooks project, consistent styling across all chapters and components enhances visual appeal and improves user experience. This guide serves as a comprehensive reference for styled components used in our Pressbooks projects. It helps student developers in the OER Lab to maintain visual harmony across the book by outlining key design principles and reusable styled components.

This article establishes the visual standards for our Pressbooks projects by detailing the color palette, typography rules, component styles, and asset guidelines to ensure consistency and cohesion across all content. Technical pressbook settings such as theme options and plugin configurations are covered in the [Getting Started with Pressbooks: A Guide for Student Developers.](/blog/pressbooks-getting-started-student)

## Our Approach

To ensure a consistent and accessible design across Pressbooks project, this guide is presented in two complmentary parts:

1. **This Blog Post** - This post serves as the rulebook for our design system. It outlines the core principles behind our visual design and sets the standards for typography, color usage, and component styling. You'll find detailed guidance on how to apply these standards, along with the CSS needed to replicate each styled component featured in the Pressbooks Template.
2. **Pressbooks Template** - Acting as a visual companion to this blog post, a template book displays all styled components in a live Pressbooks environment. It provides a clear, practical view of how each element should look and function. It also serves as the source of the custom CSS, making it easy to copy and implement styles into your own projects.

   **\*Note:** Please do not clone the Pressbooks Template directly. Instead, import only the components and CSS you need. Use both the blog post and reference book together as a guide for proper implementation.

## Styled Components

This section highlights the custom-built CSS styled components used throughout our Pressbooks projects. Each example shown here is a visual snapshot taken directly from the Reference Pressbooks Book.

**\*Note:** The appearance you see is not default—it is achieved by applying the Custom CSS provided in the reference book.

### Learning Objectives

The Learning Objectives textbox is a styled component designed to highlight the expected learning outcomes for readers. It is typically placed at the beginning of a Part (rather than individual Chapters) to outline what readers should be able to understand or achieve after completing the entire Part—including all chapters within it.

**\*Note:** In Pressbooks, content is organized hierarchically into Parts and Chapters. Each Part acts as a broader section that contains multiple related Chapters. For example, under Part 1, you might have:

- Chapter 1.1
- Chapter 1.2
- Chapter 1.3

![learning-objectives textbox](/look-and-feel-guide/learning-objectives.png)

```
.textbox .textbox--learning-objectives {
  background-color: #fff;
  border-radius: 0.5rem;
  box-shadow: 0 0 1rem rgba(0, 0, 0, 0.2);
  overflow: hidden;
  padding: 0;
  margin-top: 0.9em;
  margin-bottom: 1.35em;
  .textbox__header {
    background-color: #d4df38;
    padding: 1.7777777778em 1.7777777778em 1.7777777778em 1.7777777778em;
  }
  .textbox__content {
    margin: 0;
    padding: 1.7777777778em 1.7777777778em 1.7777777778em 1.7777777778em;
  }
}
```

### Exercises

The Exercises textbox is used to present practice activities or questions within a chapter. To maintain a cohesive and clean design, we use the `.textbox--border-left` class. However, `textbox--exercises` class is also available by default, and this class has a similar look of the learning objectives textbox which has a colored header and a content field.

![exercises textbox](/look-and-feel-guide/exercises.png)

```
.textbox .textbox--border-left {
  border-radius: 0.5rem;
  border-left-width: 8px;
  border-color: #004c97;
  background-color: #fff;
  padding: 2.5em 2.5em 2.5em 2.5em;
  margin-top: 0.9em;
  margin-bottom: 0.9em;
}
```

### Examples

The Examples textbox is used to present practical examples within a chapter. To maintain a cohesive and clean design, we use the `.textbox--border-left third-option` class. However, `textbox--examples` class is also available by default, and this class has a similar look of the learning objectives textbox which has a colored header and a content field.

![examples textbox](/look-and-feel-guide/examples.png)

### Challenge Questions

### Textbox

The General Textbox is used to highlight supplementary or important content that should be visually separated from the main body text. We’ve created three types of textboxes for consistent content highlighting. Each textbox shares the same structural styling—such as padding, border-radius, and typography—but differs only in background color to visually distinguish its purpose.

![general textbox](/look-and-feel-guide/textbox-1.png)

```
.textbox {
  font-family: "Encode Sans", sans-serif;
  padding: 2.5em 2.5em 2.5em 2.5em;
  margin: 0.9em 0 0.9em 0;
  border-style: solid;
  border-width: 0.5em;
  border-color: #003180;
  border-radius: 0;
  background-color: white;
}
```

![general textbox second option](/look-and-feel-guide/textbox-2.png)

```
.textbox .textbox--second-option {
  background-color: #dedede;
}
```

![general textbox third option](/look-and-feel-guide/textbox-3.png)

```
.textbox .textbox--third-option {
  background-color: #f0f6ff;
}
```

### Blur Box for Answers

The Blur Box component is used to present practice questions at the end of each chapter, where the answer is intentionally hidden by default. When a reader hovers their mouse pointer over the answer area, the blurred effect is removed, and the answer becomes visible.

![blur box component where answer is hidden](/look-and-feel-guide/blurbox-1.png)

![blur box component where answer is shown](/look-and-feel-guide/blurbox-2.png)

```
.answer-box {
  margin: 2rem 0;
  display: flex;
  align-items: center;
  width: fit-content;
  padding: 1rem;
  border-radius: 0.375rem;
  box-shadow: 0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -2px rgba(0, 0, 0, 0.1);
  .answer-box--answer {
    cursor: pointer;
    filter: blur(8px);
    transition: all cubic-bezier(0.4, 0, 0.2, 1) 300ms;
    &:hover {
      filter: blur(0);
    }
  }
}
```

### Accordion

The Accordion Menu is an interactive component used to hide and reveal content on click. It is used for "Text version(s) of the Question and Answers" in the H5P activities and "How to Access the Transcript" in the video component.

![accordion menu not opened](/look-and-feel-guide/accordion-1.png)

![accordion menu opened](/look-and-feel-guide/accordion-2.png)

```
details > summary {
  padding: 10px;
  font-size: 1rem;
  font-weight: bold;
  background-color: #f0f0f0;
  cursor: pointer;
  border-radius: 0.5rem;
}

details[open] > summary {
  background-color: #f79750;
  color: #000;
}

details ul {
  background-color: #f0f0f0;
  border-radius: 0.5rem;
  padding: 1.5rem;
  li {
    list-style: none;
    margin: 0;
    span {
      padding-left: 0.5rem;
      font-weight: bold;
    }
  }
}

@keyframes details-show {
  from {
    opacity: 0;
    transform: var(--details-translate, translateY(-0.5rem));
  }
}

details[open] > *:not(summary) {
  animation: details-show 750ms ease-in-out;
}
```

#### Textbox with Icons

The Textbox with Icons component has two style variations. `.textbox--header-icons` class displays an icon next to the heading text at the top of the box, and `textbox--icons` class places the icon on the left side of the box and the text content on the right.

```

    background-color: #eae8d7;

    align-items: center;
    padding: 1.78rem;
    h2 {
      color: #000;
```
