## Why a Consistent Look & Feel Matters

A unified and consistent Look and Feel is essential for creating a cohesive and professional digital product. In Pressbooks project, consistent styling across all chapters and components enhances visual appeal and improves user experience. This guide serves as a comprehensive reference for styled components used in our Pressbooks projects. It helps student developers in the OER Lab to maintain visual harmony across the book by outlining key design principles and reusable styled components.

This article establishes the visual standards for our Pressbooks projects by detailing the color palette, typography rules, component styles, and asset guidelines to ensure consistency and cohesion across all content. Technical pressbook settings such as theme options and plugin configurations are covered in the [Getting Started with Pressbooks: A Guide for Student Developers.](/blog/pressbooks-getting-started-student)

## Our Approach

To ensure a consistent and accessible design across Pressbooks project, this guide is presented in two complmentary parts:

1. **This Blog Post** - This post serves as the rulebook for our design system. It outlines the core principles behind our visual design and sets the standards for typography, color usage, and component styling. You'll find detailed guidance on how to apply these standards, along with the CSS needed to replicate each styled component featured in the Pressbooks Template.
2. **Pressbooks Template** - Acting as a visual companion to this blog post, a template book displays all styled components in a live Pressbooks environment. It provides a clear, practical view of how each element should look and function. It also serves as the source of the custom CSS, making it easy to copy and implement styles into your own projects.

   **\*Note:** Please do not clone the Pressbooks Template directly. Instead, import only the components and CSS you need. Use both the blog post and reference book together as a guide for proper implementation.

## Styled Components

This section highlights the custom-built CSS styled components used throughout our Pressbooks projects. Each example shown here is a visual snapshot taken directly from the Reference Pressbooks Book.

**\*Note:** The appearance you see is not default—it is achieved by applying the Custom CSS provided in the reference book.

### Learning Objectives

The Learning Objectives textbox is a styled component designed to highlight the expected learning outcomes for readers. It is typically placed at the beginning of a Part (rather than individual Chapters) to outline what readers should be able to understand or achieve after completing the entire Part—including all chapters within it.

**\*Note:** In Pressbooks, content is organized hierarchically into Parts and Chapters. Each Part acts as a broader section that contains multiple related Chapters. For example, you might have:

- Part 1
  - Chapter 1.1
  - Chapter 1.2
  - Chapter 1.3

![learning-objectives textbox](/look-and-feel-guide/learning-objectives.png)

```
.textbox.textbox--learning-objectives,
.bcc-box.learning-objectives {
  background-color: #fff;
  border-radius: 0.5rem;
  box-shadow: 0 0 1rem rgba(0, 0, 0, 0.2);
  overflow: hidden;
  .textbox__header {
    background-color: #d4df38;
  }
}
```

The Blur Box component is used to present practice questions at the end of each chapter, where the answer is intentionally hidden by default. When a reader hovers their mouse pointer over the answer area, the blurred effect is removed, and the answer becomes visible.

![blur box component where answer is hidden](/look-and-feel-guide/blurbox-1.png)

![blur box component where answer is shown](/look-and-feel-guide/blurbox-2.png)

```

  display: flex;
  align-items: center;
  width: fit-content;
  padding: 1rem;
  border-radius: 0.375rem;
  box-shadow: 0 4px 6px -1px rgba(0, 0, 0, 0.1),
```

### Table

The Table component is styled to create a bordered and horizontally centered table layout. Text contents in the table headers (`<th>`) are center-aligned and when the content inside table cells begins to shrink too much based on screen size, a horizontal scroll bar automatically appears.

![table component](/look-and-feel-guide/table.png)

```
.table-wrapper {
  overflow-x: auto;
  width: 100%;
  table {
    width: 100%;
    table-layout: auto;
    th {
      text-align: center !important;
    }
  }
}
```

**\*Note:** The Malala theme in Pressbooks provides several built-in CSS classes that influence table appearance by default:

- `table.grid`: Adds a black border around all table cells, using `border: 1px solid` and `border-color: black`.
- `table.aligncenter`: Centers the table horizontally within the page, using `margin-left: auto` and `margin-right: auto`.
- For more information on how spacing and structural alignment are handled, please refer to the [Layout & Spacing](#layout--spacing) section
- Please use these default CSS classes when creating a table, using `<table class=grid aligncenter">`

## How to Create Images and Diagrams

To maintain consistency and quality in visual elements across Pressbooks projects, we recommend using Canva for general image creation and GeoGebra for diagrams and graphs.

### Image Creation in Canva

1. Visit free image sites like [Pixabay](https://pixabay.com/), [Freepik](https://www.freepik.com/), or similar platforms. Search for suitable images and download your selection.
2. Go to [Canva](https://www.canva.com/) and log in using the Centennial College OER Lab account:
   - Email: OERLab@centennialcollege.ca
   - Password: OerLab@Centennial2021!
3. Click “Create a design” and choose "Custom Size" and enter 700 x 500 px (recommended size)
4. Click on the “Uploads” tab and upload your previously downloaded image
5. Adjust, crop, and edit as needed to fit your design goals
6. Click the “Share” button, then choose “Download”
7. Navigate to the page in your Pressbooks project where you want to insert the image
8. Click where you’d like to place the image and choose “Add Media”
9. Go to “Upload Files”, click “Select Files”, and upload your image
10. Once uploaded, click “Insert” at the bottom-right of the screen

### Diagram Creation in GeoGebra

1. Visit [GeoGebra](https://www.geogebra.org/) and click "Start Calculator"
2. Setting up the workspace
   - Go to setting -> Basic -> Set the x-axis and y-axis ratio to 1:1
   - Go to setting -> xAxis setting -> Set the distance to 1
   - Go to setting -> yAxis setting -> Set the distance to 1
3. Select "Tools" tab on the left side
4. Select "Vector" tool under "Lines" to draw primary vector
5. Select "Circular Sector" tool under "Circles" to draw an angle
6. Select "Segment" under "Lines" to draw a reference line
7. Select "Text" under "Media" to insert labels. For vector labels, use LaTeX syntax for an overhead arrow (`\overrightarrow{A} for vector A`)
   - Replace A with your vector's label
8. Select the line and set the "Line Thickness" to 8
9. For angles and its labels and lines, set the color to gray. The theta symbol (θ) should be black and bold
10. Set the reference line color to red and make it dotted line by selecting the line, and choose "Line Style" to dotted line
11. Set the position vectors color to blue
12. To hide auto-generated labels (like A, B, C, D), click the label and unselect "Show Object"
13. Use Serif font for non-LaTeX texts and set text size to medium, and make it bold
14. Export and cropping
    - Click "Hamburger" menu at the top left side.
    - Click "Export Image" and click "Download" button
    - After download the image, open the image and crop the image to one small square away from the last element for a clean and consistent layout
